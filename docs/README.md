# Docs

Project docs are split into a small number of purpose-driven areas:

- [`design-system.md`](./design-system.md) documents the current shared UI tokens, utility classes, and search commands.
- [`ideas/roadmap.md`](./ideas/roadmap.md) tracks where the repo is today and the most promising next directions.
- [`ideas/personal-site-transition.md`](./ideas/personal-site-transition.md) outlines the active plan for turning the host into a calmer personal site while keeping the playground as a distinct experiments section.
- [`ideas/host-migration-checklist.md`](./ideas/host-migration-checklist.md) turns that direction into a concrete execution checklist for `apps/host`, including route moves, shell changes, content migration, and verification.
- [`ideas/task-experience-direction.md`](./ideas/task-experience-direction.md) captures the next interaction/design direction for the task board so it becomes easier to scan, edit, and reason about.
- [`ideas/parking-lot.md`](./ideas/parking-lot.md) keeps intentionally parked ideas around without mixing them into active planning.
- [`superpowers/`](./superpowers/) holds the detailed planning and spec workstream for Superpowers. Leave that area untouched unless the task explicitly calls for it.
