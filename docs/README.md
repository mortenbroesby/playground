# Docs

Project docs are split into a small number of purpose-driven areas:

- [`design-system.md`](./design-system.md) documents the current shared UI tokens, utility classes, and search commands.
- [`ideas/roadmap.md`](./ideas/roadmap.md) tracks where the repo is today and the most promising next directions.
- [`ideas/parking-lot.md`](./ideas/parking-lot.md) keeps intentionally parked ideas around without mixing them into active planning.
- [`superpowers/`](./superpowers/) holds the detailed planning and spec workstream for Superpowers. Leave that area untouched unless the task explicitly calls for it.
