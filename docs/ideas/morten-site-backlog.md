# Morten Site Backlog Pull

Source review of [morten.broesby.dk](https://morten.broesby.dk/) to translate remaining cues into this repo's personal-site backlog.

Reviewed pages:

- [Home](https://morten.broesby.dk/)
- [Blog](https://morten.broesby.dk/blog)
- [Uses](https://morten.broesby.dk/uses)

## What the current site still does well

- The home page is very short and human: name, one-line positioning, software-engineer identity, teaching/helping, and a brief family note.
- The top-level structure is clear and stable: `Blog`, `Snippets`, `Uses`.
- The writing mix is broader than the current repo backlog implies: architecture, frontend, website-building, games, rate limiting, and a personal post.
- The `Uses` page is practical and specific, not aspirational. It includes editor, extensions, software, hardware, cloud services, philosophies, and inspirations.
- The older site includes a photo and a lightweight "currently listening" / presence widget, which adds personality without turning the page into a dashboard.

## What is already covered in this repo

- A calmer personal home page and separate playground model.
- `Writing` as a first-class section.
- `Uses` as a public route.
- Space for broader writing topics beyond strict frontend architecture.

## Remaining gaps worth carrying forward

### Information architecture

- The older site has a distinct `Snippets` surface. This repo has `Writing`, but no decision yet on whether short-form notes or snippets should live inside writing or as a separate route.
- The older site uses `Blog` as a clearer label for long-form posts. This repo currently uses `Writing`, which is fine, but the short-form taxonomy remains unresolved.

### Homepage personality

- The older site is more personal immediately: photo, direct self-description, and explicit teaching/helping language.
- The current repo home page is intentionally spare, but it could still borrow a little more warmth and presence without becoming busy.

### Uses depth

- The existing repo `Uses` surface is calmer and more editorial, but it is less exhaustive than the older site.
- The old `Uses` page has strong concrete categories that could be reintroduced selectively:
  - editor and terminal stack
  - notable extensions and tools
  - software utilities
  - hardware and desk tech
  - cloud services
  - inspiration list

### Writing breadth

- The older blog proves there is room for:
  - architecture and design-process essays
  - frontend topic clusters
  - personal website and build notes
  - games-for-programmers pieces
  - occasional personal writing
- The current repo backlog should keep that breadth visible so writing does not collapse into only one technical tone.

## Backlog additions promoted from this review

- Add a decision item for `Snippets` versus `Writing` taxonomy.
- Add a lighter "presence" enhancement idea for the home page, such as photo, current note, or now-playing-style module.
- Add a `Uses` enrichment pass that selectively restores concrete categories from the older site.

## Default recommendations

- Keep `Writing` as the canonical long-form label in this repo.
- Do not add a separate `Snippets` route yet.
- Instead, add a backlog item to decide whether snippets should be:
  - a subtype inside `Writing`, or
  - a separate public route later if volume justifies it
- Keep the home page minimal, but allow one more personal signal beyond text.
- Treat the old `Uses` page as a source of category depth, not a blueprint to recreate verbatim.
