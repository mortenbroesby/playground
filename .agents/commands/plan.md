---
description: Break work into small verifiable tasks
---

Use the `engineering-workflow` and `planning-and-task-breakdown` skills.

Create an implementation plan:

1. Read the relevant spec, request, repo rules, and local architecture context.
2. Split work into ordered vertical slices with acceptance criteria.
3. Mark dependencies and any parallelizable side tasks.
4. Include the narrowest useful verification for each slice.
5. Keep the plan lightweight unless the user asks for a full planning artifact.

Do not edit code while producing a plan unless the user explicitly asks for
implementation too.
