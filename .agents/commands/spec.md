---
description: Define what to build before coding
---

Use the `engineering-workflow` and `spec-driven-development` skills.

Write or refine a compact spec before broad implementation:

1. Clarify objective, target user, constraints, non-goals, and acceptance
   criteria.
2. Identify repo-specific boundaries from `AGENTS.md`, `.agents/rules/`, and
   relevant vault notes.
3. Name the verification commands or manual checks that will prove completion.
4. For small tasks, keep the spec in the conversation. For durable or larger
   work, update the appropriate repo doc or vault note.

Do not start implementation until the spec is clear enough to test against.
